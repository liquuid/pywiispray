\section{Introdução}
\subsection{Nascimento do Python}

\begin{frame}
  \frametitle{Onde e Quando}
  \begin{itemize}
  \item Criada em 1989 pelo holandês \textbf{Guido van Rossum} no
    \emph{Centrum voor Wiskunde en Informatica} (CWI), em Amsterdã,
    Holanda.
  \item Influenciada pela linguagem ABC, desenvolvida no CWI por Guido
    e outros nas décadas de 70 e 80. ABC tinha um foco bem definido:
    ser uma linguagem de
    programação para usuários inteligentes de computadores que não
    eram programadores: Físicos, Cientistas Sociais e até Linguistas.
  \item O projeto de sistema operacional distribuído Amoeba precisava
    de uma linguagem de \emph{script}. Nasce o Python.
  \end{itemize}
\end{frame}

\begin{frame}
  \frametitle{Bases do Python}
  \begin{itemize}
  \item Elementos que eram bem sucedidos no ABC.
  \item Estruturas de dados poderosas inclusas: Listas,
    Dicionários, \emph{Strings}.
  \item Usar indentação para delimitar blocos, eliminando chaves.
  \item Fácil extensão (lição aprendida com os erros do ABC).
  \item Fácil de portar: além do Amoeba, gostaria de executar em
    Unix, Macintosh e Windows.
  \item Influências de Modula-2 e Modula-3: módulos e \emph{namespaces}.
  \end{itemize}
\end{frame}

\begin{frame}
  \frametitle{Ambiente de Concepção}

  \begin{itemize}
  \item \textbf{Universidade:} pessoas altamente especializadas para
    desenvolver e opinar sobre os elementos do projeto
  \item \textbf{Descontraído:} o nome Python vem da série de humor
  \textit{Monty Python's Flying Circus}
  \item \textbf{Sem prazos, Sem pressão:} o desenvolvimento não foi
    pressionado por estratégias de marketing, prazos, clientes ou
    qualquer outro fator que pudesse influenciar nas decisões de
    projeto, \textbf{resultando em maior qualidade.}
  \item \textbf{Software Livre:} garante a vida da tecnologia.
  \end{itemize}
\end{frame}


\subsection{Sobre a Linguagem}
\begin{frame}
  \frametitle{Características Básicas}
  \begin{itemize}
  \item \textbf{Interpretada:} usa máquina virtual, facilita
    portabilidade.
  \item \textbf{Interativa:} pode-se programar interativamente, os
    comandos são executados enquanto são digitados.Por exemplo usando o ipython que facilita testes,
    desenvolvimento rápido e outros. Facilitadores estão presentes
    \lstinline|help(obj)|.
  \item \textbf{Orientada a Objetos:} tudo\footnote{Quase tudo é um
    objeto :-)} é objeto: números, \emph{strings}, funções, classes,
    instâncias, métodos, ...
  \item \textbf{Fortemente Tipada:} Não existe \emph{casts} e nem
    conversão automática, não se mistura tipos
    ``automagicamente''.
  \item \textbf{Tipagem Dinâmica:} A tipagem de um objeto é feita em
    tempo de execução. Um objeto tem tipo, uma variável não.
  \end{itemize}
\end{frame}


\begin{frame}
  \frametitle{Características Importantes}
  \begin{itemize}
  \item Sintaxe clara, sem caracteres ``inúteis'':
    \begin{itemize}
      \item blocos são marcados por indentação
      \item parênteses são opcionais, só precisam ser utilizados para
	eliminar ambiguidades.
      \item palavras-chave (\emph{keywords}) e formações que ajudam na
	leitura, como \lstinline|for ... in ...|.
    \end{itemize}

  \item Fácil extensão: codificar nos módulos é muito fácil, podendo
    utilizar bibliotecas nativas, aproveitando desempenho,
    características nativas das plataformas, etc.
    \begin{itemize}
    \item API Python/C é bem simples
    \item Diversos conversores automáticos (SWIG, SIP, ...)
    \item \textbf{Jython:} usando Python em Java e vice-versa.
    \item \textbf{PyRex:} pseudo linguagem para facilitar integração
      Python + C/C++.
    \end{itemize}
  \end{itemize}
\end{frame}


\begin{frame}
  \frametitle{Características Importantes (2)}
  \begin{itemize}
  \item \textbf{Tipos básicos poderosos:} listas, dicionários
    (\emph{hash tables}), \emph{strings}, ... otimizados e de fácil
    uso.
  \item \textbf{Baterias Inclusas:} biblioteca padrão contém diversos
    recursos úteis: Interface Gráfica (Tk), XML, Servidores (TCP, UDP,
    HTTP, ...), HTML, protocolos de internet (email, http, ...),
    xmlrpc, ...
  \item Grande base de código e bibliotecas de terceiros
  \item Grande comunidade de desenvolvedores
  \item \textbf{Software Livre:} liberdade de uso,
    distribuição. Licença própria, compatível com GPL, porém pode
    distribuir somente o binário.
  \item \textbf{Independente:} a entidade sem fins lucrativos Python
    Software Foundation cuida da propriedade intelectual do Python.
  \end{itemize}
\end{frame}

\subsection{Opiniões sobre o Python e Comparações}

\begin{frame}
  \frametitle{Bruce Eckel}

  Bruce Eckel é o autor de livros de renome, como ``Thinking in Java''
  e ``Thinking in C++'', mas hoje prefere pensar em Python. Ele
  apresentou 10 razões por que ele ama esta linguagem, algumas delas:

  \begin{itemize}
  \item \textbf{Python is about me}
    \begin{itemize}
    \item Outras linguagens: \textit{``Sim, nós estamos tentando fazer sua
        vida mais fácil com esta linguagem, mas estas coisas são mais
        importantes''}.
    \item Python: \textit{``Nós tentamos fazer sua vida mais fácil, e
        é isto. Fazer sua vida mais fácil é o que não vamos
        comprometer.''}.
    \item \textbf{Exemplo:} C++ tentou fazer a vida do desenvovedor
      mais fácil, porém comprometimentos com performance e
      compatibilidade com C++, o que não ajudou em muito a vida do
      desenvolvedor. Qualquer problema do C++ pode ser explicado
      devido a estas duas características.
    \item \textbf{Exemplo:} Java achou que o marketing era mais importante.
    \end{itemize}
  \end{itemize}
\end{frame}


\begin{frame}
  \frametitle{Bruce Eckel (2)}
  \vspace{-1em}

  \begin{itemize}
  \item \textbf{Menos Porcaria:} Dizem que uma pessoa armazena $7 \pm
    2$ informações ao mesmo tempo, quanto menos porcaria, melhor.
    \begin{itemize}
    \item \textbf{Java:} 3 passos para abrir um arquivo. Muito mais para iterar
      sobre suas linhas.
    \item \textbf{Python:} 1 chamada para abrir um
      arquivo. Construções fáceis para iterar sobre o conteudo:
      \lstinline|for line in file('fname').readlines(): print line|.
    \item A falta de caracteres inúteis e a obrigatoriedade de uma
      identação consistente também ajuda a se concentrar no problema.
    \end{itemize}

  \item \textbf{Produtividade é mais importante que performance:}
    otimizar prematuramente é um grande erro.
    \begin{itemize}
    \item A maioria dos programas não precisa de alta-performance
    \item A maioria dos programas precisa de alta-produtividade
    \item Dos programas que precisam de performance, a maioria se
      resolve com um \textit{profiler} e otimizações algorítmicas.
    \end{itemize}
  \end{itemize}
\end{frame}
